- [x] Step 5: Build scoring engine
  - Implemented scoring logic used by `query`:
    - Confidence:
      - `0.95` when coverage rows for the test/artefact pair include covered hits.
      - `0.40–0.45` when coverage exists for the commit but the pair is not covered.
      - `0.60` for static/manual linkage when no coverage has been ingested.
    - Strength:
      - `strength = classification_weight / fan_out`
      - weights: `unit=1.0`, `integration=0.7`, `e2e=0.4`
  - Added fan-out fallback from `test_links` when classification fan-out is unavailable.

    - Computes verification summary (`untested` / `partially_tested` / `well_tested`).
    - Outputs PRD-aligned JSON payload to stdout.

  - Edge cases now handled:
